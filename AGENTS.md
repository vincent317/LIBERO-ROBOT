# Repository Instructions

When editing the GitHub Pages site under `docs/`, read and follow
`docs/PAGES_TEMPLATE.md` first.

The Pages site should keep the clean case-study template:

- Overview of dataset, task family, and environment.
- Separate `Single-task policy` and `Multi-task policy` sections.
- Standalone thinking-process blocks inside both sections.
- Statistics separated from thinking process.
- Before/after video comparisons with truthful labels and high-resolution clips.

Do not replace this with a minimal summary, timeline-heavy page, or marketing-style landing page.
